TASK 01

社群分享 OG 圖設定

Josh 提供的品牌橫幅圖（RDlogo.jpg）品質佳、識別度高，直接用作全站預設 OG 圖。
設定完成後，reasondesign.com.tw 的連結被分享在 LINE、FB、IG 時，都會顯示這張圖。

✅ 素材狀態：RDlogo.jpg 已收到，可直接上傳使用。無需再製作。
1
上傳圖片到 WordPress 媒體庫
WordPress 後台 → 媒體 → 新增媒體 → 上傳 RDlogo.jpg
上傳後點選圖片，複製右側「檔案 URL」備用。
2
設定 Rank Math 全站預設社群圖
Rank Math → Titles & Meta → 首頁（Home Page）→ 上方分頁切到「Social」→
「Facebook 縮圖」欄位 → 選擇媒體 → 選剛才上傳的 RDlogo.jpg → 儲存。
3
設定 Rank Math 全域備用圖（其他頁面沒有自訂圖時自動套用）
Rank Math → General Settings → Links → 往下找「Default OpenGraph Thumbnail」→
選同一張 RDlogo.jpg → 儲存變更。
4
驗證（30 秒）
開新分頁，進 Facebook 分享偵錯工具（無需登入）→ 貼上 https://reasondesign.com.tw → 點「偵錯」→ 確認「預覽圖片」顯示品牌圖即完成。
TASK 02

首頁 FAQ — 問答內容

共 7 題，每題自然帶入核心關鍵字。內容採「業主語言」撰寫，Google AI Overview 可直接引用。

若善設計提供哪些室內設計服務？
若善設計以台中為主要服務據點，提供住宅設計、新成屋裝潢、預售屋客變、老屋翻新及商業空間設計等完整服務。我們從業主真實的生活需求出發，以現代簡約為設計核心，為每位客戶打造獨一無二的居住空間。
什麼是預售屋客變？若善設計可以協助嗎？
預售屋客變是指在預售屋完工前，向建商申請變更室內格局、開口位置或建材規格的流程。由於客變有時間限制，且完工後再修改成本大幅增加，建議在交屋前及早與設計師討論。若善設計提供完整的預售屋客變服務，協助業主在時間壓力內做出最適合長期生活的格局決策。
老屋翻新和新成屋裝潢，流程有什麼不同？
老屋翻新需要在設計前進行結構評估與現況勘察，釐清管線是否需要更新、牆面是否需要補強，施工期通常比新成屋略長。新成屋裝潢則從乾淨的毛坯屋出發，設計自由度更高，可從格局、動線到建材風格全面規劃。兩種類型若善設計都有豐富的執行經驗，會依實際屋況提供最適合的建議。
台中室內設計費用大概是多少？
室內設計費用因坪數、設計複雜度、建材品質和工程範圍不同而有所差異。若善設計採透明報價，從初步洽談到提案階段，我們會依照您的需求與預算提供詳細的費用估算。建議直接預約免費初談，讓設計師了解您的空間與需求後，提供最精準的報價。
若善設計的設計風格是什麼？
若善設計以現代簡約風格為核心，強調空間秩序、光線運用與建材質感的搭配。我們不追求單一風格標籤，而是從每位業主的生活習慣與審美出發，讓空間語言回應真實的居住需求。若善的作品曾榮獲德國 iF Design Award 與美國 MUSE 金獎肯定。
從設計到完工需要多少時間？
一般住宅設計案從洽談到完工約 4–8 個月，依空間坪數與工程範圍調整。若善設計的標準流程分為業務洽談、初步提案、設計合約、工程估驗、裝修施工、完工驗收六個階段，每個階段都有明確的時間節點，讓業主隨時掌握進度。
若善設計的服務範圍有哪些地區？
若善設計以台中市為主要服務範圍，工作室位於台中市西區東興路三段 160 號 5 樓。外縣市案件可依個案討論，歡迎先透過電話 04-2328-1088 或 Email service@reasondesign.com.tw 與我們聯繫。
💡 關鍵字覆蓋確認：預售屋客變、老屋翻新、新成屋裝潢、台中室內設計費用、現代簡約、台中室內設計、設計流程 — 全部自然帶入，不生硬。

TASK 02 — WP

WordPress 操作步驟

WordPress 加入 FAQ 的方式有兩種，擇一執行即可：

推薦方式（Rank Math 原生 Schema） — Rank Math 有內建 FAQ Schema 功能，不需貼程式碼，UI 直接填問答即可，Google 自動識別。

方式 A：Rank Math 原生 FAQ Schema（推薦）

1
進入首頁編輯
WordPress 後台 → 頁面 → 找到首頁 → 編輯（Elementor 或 Block Editor）
2
開啟 Rank Math Schema
右上角找 Rank Math 圖示（或 Block Editor 右側欄）→ 點「Schema」分頁 → 「Add Schema」→ 選「FAQ」
3
填入問答
把上方 7 組 Q&A 一題一題填入 FAQ Schema 的欄位 → 儲存頁面
4
在頁面加入 FAQ 可見區塊
在 Elementor 中，頁面適當位置（建議在作品集下方、聯絡區塊上方）加入 Accordion 或 Toggle 元件，把同樣 7 題填入作為可見的 FAQ 區塊。Rank Math Schema 和頁面可見內容分開，都要做。

方式 B：貼入 JSON-LD 代碼（備用）

1
Rank Math → General Settings → Analytics → 往下找「Header Code」或安裝 WPCode 插件
推薦用 WPCode（免費）或 Rank Math 的 Custom Schema 功能，不要直接改 functions.php
2
貼入上方 Schema 代碼
設定為只在「首頁」執行，不要套全站（以免和其他頁面的 Schema 衝突）
✅ 完成後驗證：進 Google Rich Results Test → 貼上首頁網址 → 應看到「FAQPage」Schema 通過，可展開顯示 7 題問答。
